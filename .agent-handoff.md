## CONTEXTO

Refactor del admin panel para multi-tenancy con JWT, roles, admin v2, wizard provisioning, templates, métricas y auditoría.

**Hecho en esta sesión:**

- Backend: `POST /auth/login|refresh|logout`, `src/services/auth.ts` (bcrypt + JWT), `src/middleware/jwt.ts`, `adminAuth` acepta **Bearer JWT** o **X-Admin-Secret** (deprecated log), `bootstrapSuperAdmin` al arranque + en login.
- `GET /admin/v2/me`, `/admin/v2/tenants`, nested `/admin/v2/tenants/:tenant_id/{users,wabas,apps,templates,messages}`, `/admin/v2/audit-log`.
- **Orden de rutas:** el router `tenants/:tenant_id` está montado **antes** de `tenants` para que matchee `/tenants/:id/messages` (crítico).
- Onboard: `requireOnboardTenantScope` + status session filtrado por tenant para JWT.
- Frontend: `AuthProvider`, login email/password, tokens en `localStorage`, `api.ts` con refresh en 401, rutas `/super/*` y `/dashboard/*`, wizard baseline `ProvisionNumber`, templates JSON + lista Meta, métricas simples en `Home`, auditoría tabla.
- Tests: `auth.test.ts`, `admin-v2-tenants.test.ts`, `admin-v2-isolation.test.ts`; `vitest.setup.ts` con JWT secrets default para CI.
- `CRITICAL_ENV` incluye `JWT_ACCESS_SECRET` y `JWT_REFRESH_SECRET` (breaking: hay que setearlos en Railway/prod).
